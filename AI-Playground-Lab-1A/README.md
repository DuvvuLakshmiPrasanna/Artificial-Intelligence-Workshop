## Key Takeaways

1. **Accuracy ≠ compliance.** Claude preserved every key number in Task 1 but broke the 15-word/bullet rule. A tool can be more accurate and less instruction-following at the same time.

2. **"Works" is not the same as "correct."** Gemini's Task 2 code ran without errors but counted stop words like "the" and "a" as missing skills — a logic flaw invisible until the output was tested.

3. **All four tools solved the reasoning puzzle correctly.** Differentiation for simple arithmetic is in *how transparently* they show work, not whether they get the answer right.

4. **Know your tool's default mode.** Perplexity added inline citations to a summarisation task that didn't ask for them — because it is a search-first tool by design.

5. **The verification chain matters.** Frontier AI models are wrong 8–12% of the time on facts. The habit is: AI first → secondary source → primary URL. This reduced student factual errors by 60% in the article studied.

---

## Verification Chain (to use with students)

```
Step 1 → Ask the AI tool
Step 2 → Verify with Perplexity (cites primary sources)
Step 3 → Open the primary source URL and confirm the number / date / fact yourself
```
